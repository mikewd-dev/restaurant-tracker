# Restaurant Tracker – Case Study

## Overview
Restaurant Tracker is a web application that helps users discover restaurants in London. It integrates **Mapbox** for a dynamic, interactive map interface and uses the **Local Business API** to fetch restaurant details.

This project was a **self-assigned group project during a bootcamp**, where our team planned the idea, designed the solution, and implemented the features from scratch. The goal was to create a **real-world, interactive, and responsive web application** that could be showcased to potential employers.

---

## My Contributions

- Built the **frontend search and filtering functionality** using Bootstrap and jQuery.  
- Integrated **Mapbox maps** to display restaurant locations dynamically.  
- Connected to the **Local Business API** to fetch restaurant data and render it on the map.  
- Implemented **client-side storage** to save user preferences and visited restaurants.  
- Collaborated using **Git branches, pull requests, and GitHub Issues** to manage tasks and track progress.  
- Ensured the application is **responsive, interactive, and polished** for end-users.

---

## Features

- Interactive map showing restaurants in the user’s area.  
- Search and filter restaurants by type or name.  
- Persistent client-side storage for user selections.  
- Fully responsive UI built with Bootstrap.  
- Integration with multiple server-side APIs for real-time data.
